This is the **"Shielded Master Prompt."** It has been reinforced with specific technical directives to prevent the "confused intern" behavior, ensuring the AI handles unit scanning, ID scoping, and action sub-parsing correctly from the first line of code.

### **The Shielded Master Prompt: Project "CoreUI"**

**Task:** Build "CoreUI"—a Go-based DSL compiler that parses `.cui` files into a deterministic JSON schema. 

#### **1. Project Constraints & Environment**
* **Strict Zero-Dependency:** Use ONLY the Go Standard Library. No `go get`, no `testify`, no external modules.
* **Architecture:** Lexer -> Parser -> Registry-based Validator -> JSON Generator.
* **Documentation:** Provide `PURPOSE.md`, `SPEC.md`, and `instructions.md` as drafted in our blueprint.

#### **2. Lexer Implementation (The Unit Fix)**
* **The Unit Trap:** Standard `text/scanner` breaks on units like `10px`. 
* **Directive:** Implement a custom scanning wrapper. When a number is encountered, peek at the next characters. If they match `px`, `%`, `*`, or `auto`, consume them as a single **Unit Token**. Do not allow a space between the number and the unit.

#### **3. Registry & Validation (The Firewall)**
* **Registry-First:** Implement `pkg/registry` as the source of truth for all components (View, Stack, Grid, Box, Text, Input, Trigger, DataTable).
* **Validation Engine:** The parser must call `registry.IsAttributeAllowed(component, attr)` for every property. Throw a fatal error with line/column numbers for any unknown attribute or type mismatch.

#### **4. Parser Logic (Identity & Actions)**
* **Global ID Scope:** Maintain a `map[string]bool` in the Parser state. Every component **must** have an `id`. If an ID is missing or has already been used anywhere in the file, throw a `Duplicate/Missing ID` error.
* **Action Sub-Parsing:** To avoid "Quote Inception," treat the value of the `action` attribute as an opaque string during the main parse. Pass this string to a dedicated `ActionParser` that decomposes it into `namespace`, `call`, and `params`.
* **Built-in UI Namespace:** Explicitly support `ui:navigate(target)`, `ui:toggle(id)`, and `ui:close()`.

#### **5. Output Requirements (The Enhanced Index)**
Generate a JSON file with the following root keys:
1.  **`tree`**: The recursive AST.
2.  **`index`**: A flat map where `key = component_id`. The value must be an object: `{ "path": "/json/path", "type": "ComponentType" }`.
3.  **`metadata`**: Compilation timestamp and version.

#### **6. Test Suite (The Golden File Mandate)**
* **Negative Tests:** Create `parser_test.go` with cases that intentionally fail (duplicate IDs, invalid properties for a specific component, malformed action strings).
* **Golden Files:** Create a `/testdata` directory. For every `.cui` example, provide a corresponding `.json` file. The test suite must compile the `.cui` and perform a deep comparison against the JSON "Golden File."

#### **7. Execution Plan**
1. Initialize the Go module.
2. Define the `Registry` and `AST` structures.
3. Build the Lexer with the "Unit" peek logic.
4. Build the Recursive-Descent Parser with the Global ID map.
5. Implement the CLI in `cmd/corec`.
6. Run the Test Suite to verify the "Firewall" is working.
